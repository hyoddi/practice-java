## 알고 들어가기 : Comparator

정렬 기준을 직접 정의하는 인터페이스

**애 자체가 정렬을 해주는 것이 아니라, 두 원소를 비교해서 "누가 앞에 와야 하는지"만 알려준다**

```java
int compare(T o1, T o2);
```

- **음수**→ 첫 번째(o1)가 앞
- **0** → 순서 유지 (두 원소를 같은 순서로 취급)
- **양수** → 두 번째(o2)가 앞

실제로 `-1`, `0`, `1`을 직접 반환할 수도 있지만, 보통은 두 값을 비교한 결과를 반환

```java
// 1. 이건 람다식으로 짧게 쓴거
(a, b) -> Integer.compare(a, b)
// a와 b를 받아서 Integer.compare(a,b)를 반환
// Comparator 인터페이스에는 메서드가 저거 하나라 알아서 자바가 이해한다네요

// 2. 1과 같은 식
new Comparator<Integer>() {
    @Override
    public int compare(Integer a, Integer b) {
        return Integer.compare(a, b);
    }
}
```

```java
// 오름차순
(a, b) -> Integer.compare(a, b)

// 내림차순
(a, b) -> Integer.compare(b, a)
```

## 문제

문자열로 구성된 리스트 strings와, 정수 n이 주어졌을 때, 각 문자열의 인덱스 n번째 글자를 기준으로 오름차순 정렬하려 합니다. 예를 들어 strings가 ["sun", "bed", "car"]이고 n이 1이면 각 단어의 인덱스 1의 문자 "u", "e", "a"로 strings를 정렬합니다.

#### **제한 조건**

- strings는 길이 1 이상, 50이하인 배열입니다.
- strings의 원소는 소문자 알파벳으로 이루어져 있습니다.
- strings의 원소는 길이 1 이상, 100이하인 문자열입니다.
- 모든 strings의 원소의 길이는 n보다 큽니다.
- 인덱스 1의 문자가 같은 문자열이 여럿 일 경우, 사전순으로 앞선 문자열이 앞쪽에 위치합니다.

#### 입출력 예

| strings | n | return |
| --- | --- | --- |
| ["sun", "bed", "car"] | 1 | ["car", "bed", "sun"] |
| ["abce", "abcd", "cdx"] | 2 | ["abcd", "abce", "cdx"] |

#### 입출력 예 설명

**입출력 예 1**

"sun", "bed", "car"의 1번째 인덱스 값은 각각 "u", "e", "a" 입니다. 이를 기준으로 strings를 정렬하면 ["car", "bed", "sun"] 입니다.

**입출력 예 2**

"abce"와 "abcd", "cdx"의 2번째 인덱스 값은 "c", "c", "x"입니다. 따라서 정렬 후에는 "cdx"가 가장 뒤에 위치합니다. "abce"와 "abcd"는 사전순으로 정렬하면 "abcd"가 우선하므로, 답은 ["abcd", "abce", "cdx"] 입니다.

## 풀이

```java
public class StringSort {
    public String[] solution(String[] strings, int n) {

        Arrays.sort(strings, (a, b) -> {
            // n번째 문자가 다르면 그 문자끼리 비교, 같으면 a.compareTo(b)
            if (a.charAt(n) != b.charAt(n)){
                return Character.compare(a.charAt(n), b.charAt(n));
            }
            return a.compareTo(b);
        });

        return strings;
    }
}
```

# 접근 방식

### 1. 문제에서 요구하는 정렬 기준을 찾는다

보통 정렬 문제는 다음 셋 중 하나

- 숫자 자체
- 문자열 자체
- **특정 조건(특정 인덱스, 길이, 여러 조건 등)** ← 이 문제

**→ 기본 정렬이 아니라 특정 기준으로 정렬해야 하면 Comparator를 떠올린다.**

### 2. Arrays.sort()에 Comparator를 넘길 수 있는지 확인한다

```java
Arrays.sort(배열,Comparator)
```

Comparator는 두 원소를 어떻게 비교할 것인가만 정의하면 OK

### 3. 비교 기준을 순서대로 적는다

이 문제는 조건이 두 개

```
1순위 : n번째 문자
2순위 : 사전순
```

비교 기준을 먼저 적어두면 코드가 훨씬 쉬워진다

### 4. 우선순위가 높은 조건부터 비교한다.

```java
if (1순위가 다르다) return1 순위 비교 return2 순위 비교
```

이 패턴이 Comparator의 거의 모든 문제에서 사용!!

### 5. 문자열은 compareTo(), 문자는 Character.compare()

```java
// 문자열
a.compareTo(b)

// 문자
Character.compare(a.charAt(n),b.charAt(n))
```

```java
// 굳이 직접 비교 X
if (...) return 1;
else return -1;
```

# Comparator 템플릿

```java
Arrays.sort(arr, (a, b) -> {

    // 1순위 비교
    if (조건이 다르다) {
        return 비교;
    }

    // 2순위 비교
    return 비교;
});
```
